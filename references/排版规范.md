# 排版规范

此文件定义了不同内容密度下的排版策略。生成 HTML 汇报时，每张幻灯片必须根据其内容类型选用对应的排版方案。

核心原则：**不要所有内容居中，也不要所有内容左对齐 — 按内容密度自适应。**

---

## 垂直居中补偿（通用规则）

当幻灯片内容较少、只占据视口上半部分时，**必须**让整个内容块垂直居中，避免"贴顶、底部大片留白"。
```css
/* 内容较少时垂直居中 — 适用于所有排版策略，不影响内部元素水平对齐 */
.slide-content.center-v {
    justify-content: center;
}
/* 内部元素保持自身对齐方式：section-head 继续左对齐，卡片网格继续左对齐 */
.slide-content.center-v .section-head {
    text-align: left;
}
```
### 判断标准（硬性，逐 slide 评估）
**满足以下任一条件即必须加 `center-v`：**
| 内容组合 | 示例 | center-v |
|---|---|---|
| section-head + ≤ 4 张卡片 | 工作流优势、业务可行性、团队能力 | ✅ |
| section-head + 一段话 + ≤ 5 项列表 | 项目目标、核心结论、关键指标 | ✅ |
| section-head + 1 张图 + ≤ 5 项 check-list | 最新进展、阶段成果 | ✅ |
| section-head + 1-2 张 hero 图（无其他实质内容） | 核心流程图、系统架构图展示页 | ✅ |
| 封面页 / 结束页（纯标题 + 副标题） | 标题页、结尾致谢 | ✅ |
**反例（不加 center-v 的大密度页面）：**  | 内容组合 | 示例 | center-v |
|---|---|---|
| 大表格（≥ 10 行） | 费用进度总览、测试用例覆盖 | ❌ 不写 |
| 双栏布局且两侧都密集 | 表格 + 代码块、列表 + 大表格 | ❌ 不写 |
| 多个大 hero 图堆叠 | 2+ 张架构图上下排列 | ❌ 不写 |
| 图文并排且图片很大（≥ 50vh） | 左文字、右满屏大图 | ❌ 不写 |

### 生成后逐 slide 复查
**生成完整 HTML 后，必须遍历每张 slide 检查：**
- 如果 slide 内容看起来稀疏（空白多、内容集中在上半屏），检查是否漏加了 `center-v`
- 如果 slide 内容很满（占满整个视口甚至需要滚动），确保**没有**加 `center-v`（否则会挤压变形）  注意：此规则不影响 section-head 的水平左锚点对齐方式，只改变整体的垂直对齐。

---

所有非标题页的章节开头，必须使用统一的 `section-head` 模式：
```
┌─────────────────────────────────────────────────────┐
│  01 · Kicker（章节编号+标签）                         │
│  H2 章节大标题                                       │
│  Note 说明文字（放在大标题下方，左对齐）              │
└─────────────────────────────────────────────────────┘
```
### 规范
| 要素 | 规范 |
|------|------|
| 布局 | 纵向左对齐：kicker 在上，h2 大标题在中，note 放在 h2 下方；默认不使用右栏 note |
| Kicker | 小号 13px、使用强调色（`var(--accent)`）、加粗 900、可选加左侧状态小圆点 |
| h2 标题 | 大号 36-40px、紧贴 kicker 下方（margin-top: 10px）、line-height 1.18 |
| Note 说明 | 段落文字、辅助色（`var(--muted)`）、15-17px、line-height 1.7-1.8、左对齐、宽度受控在 `min(760px, 100%)` |
| 移动端 | 保持纵向结构，不需要从双栏切换 |

### HTML 示例
```html
<section class="slide">
<div class="slide-content">
<div class="section-head reveal">
<div class="section-kicker">01 · 背景与目标</div>
<h2>项目背景与核心目标</h2>
<p class="section-note">简要说明本阶段的工作重点和预期成果。</p>
</div>
<!-- 内容正文 -->
</div>
</section>
```
### CSS
```css
.section-head {
    display: flex;
    flex-direction: column;
    align-items: flex-start;
    gap: clamp(8px, 1vw, 14px);
    margin-bottom: 34px;
    text-align: left;
}
.section-kicker {
    color: var(--accent);
    font-size: clamp(0.75rem, 1.2vw, 0.85rem);
    font-weight: 900;
    text-transform: uppercase;
}
.section-note {
    margin: 0;
    color: var(--text-secondary, var(--muted));
    font-size: clamp(0.9rem, 1.3vw, 1rem);
    line-height: 1.8;
    max-width: min(760px, 100%);
    text-align: left;
}
```

## 徽章/标签语义变体系统

任何汇报的表格或卡片中都可直接使用以下状态标签：
```css
.badge {
    display: inline-flex;
    padding: 5px 9px;
    border-radius: 999px;
    font-size: 12px;
    font-weight: 900;
    white-space: nowrap;
}
/* 5 种语义变体 — 颜色从主题变量派生，无对应变量时用统一灰度 */
.badge.ok {
    background: rgba(var(--jade-rgb, 31, 123, 98), 0.12);
    color: var(--jade, #1f7b62);
}
.badge.warn {
    background: rgba(var(--gold-rgb, 185, 139, 61), 0.16);
    color: var(--gold-dark, #805a1d);
}
.badge.danger {
    background: rgba(var(--accent-rgb, 183, 25, 43), 0.10);
    color: var(--accent, #b7192b);
}
.badge.todo {
    background: rgba(var(--muted-rgb, 98, 112, 126), 0.12);
    color: var(--muted, #62707e);
}
.badge.info {
    background: rgba(var(--blue-rgb, 35, 72, 93), 0.10);
    color: var(--blue, #23485d);
}
/* 语义说明：ok=已完成/正常 | warn=进行中/注意 | danger=阻塞/异常 | todo=待开始 | info=备注 */
```
### HTML 用法
```html
<span class="badge ok">已完成</span>
<span class="badge warn">进行中</span>
<span class="badge danger">有风险</span>
```

## 四种内容密度自适应排版策略

### 策略一：表格页
**适用场景：** 幻灯片主要内容为表格（如对照表、进度表、数据表）
**布局规则：** - h2 标题：**左锚点**（`text-align: left`）
- 表格：`width: 100%`，自然填充宽度
- 表格字号：`font-size: clamp(0.75rem, 1.3vw, 1rem)`
- 表头左对齐，用底部 2px 强调色横线分隔
- 表格内容左对齐（中文表格左对齐阅读效率最高）
- 表格前后的间距用 `var(--content-gap)`
**代码示例：**
```html
<section class="slide">
<div class="slide-content">
<h2 class="reveal">项目进度总览</h2>
<table class="reveal">
<thead>
<tr><th>模块</th><th>进度</th><th>负责人</th><th>状态</th></tr>
</thead>
<tbody>
<tr><td>需求分析</td><td>100%</td><td>张三</td><td>已完成</td></tr>
<tr><td>设计阶段</td><td>80%</td><td>李四</td><td>进行中</td></tr>
</tbody>
</table>
</div>
</section>
```
---

### 策略二：图文混排页
**适用场景：** 幻灯片同时包含文字描述和图片（如方案说明配架构图）
**布局规则：**
- h2 标题：**左锚点**
- 内容区域：左右双栏布局（`display: flex` 或 `display: grid`）
- 左栏：文字描述，宽度约 40%-50%
- 右栏：图片，宽度约 50%-60%
- 图片设置 `max-height: min(55vh, 500px)`，`object-fit: contain`
- 保持左右栏比例，不让图片过大挤占文字空间
**代码示例：**
```html
<section class="slide">
<div class="slide-content">
<h2 class="reveal">系统架构设计</h2>
<div class="two-column-layout reveal">
<div class="column-text">
<p>采用微服务架构，将核心业务模块独立部署...</p>
<ul>
<li>服务发现与注册</li>
<li>负载均衡策略</li>
<li>故障自动转移</li>
</ul>
</div>
<div class="column-image">
<img src="assets/architecture.png" alt="系统架构图" class="architecture" />
<p class="image-caption">图：系统整体架构设计</p>
</div>
</div>
</div>
</section>
```
```css
.two-column-layout {
    display: flex;
    gap: var(--content-gap);
    align-items: flex-start;
}
.column-text {
    flex: 1;
}
.column-image {
    flex: 1.2;
}
```
---

### 策略三：稀疏内容页
**适用场景：** 幻灯片只有一句话描述和一张核心图片（如核心流程图、关键节点图）
**布局规则：**
- h2 标题：**左锚点**
- 描述文字精简为一句话，字体中等大小
- **图片作为绝对主角**，撑满剩余全部空间
- 图片字号：`max-height: min(60vh, 600px)`，更大的空间赋予更重要的视觉内容
- 不要让稀疏内容留下大片空白 — 用图片填充
**代码示例：**
```html
<section class="slide">
<div class="slide-content">
<h2 class="reveal">核心流程节点</h2>
<p class="reveal summary-text">以下为该业务的核心处理流程，共分为六个关键环节。</p>
<div class="hero-image-container reveal">
<img src="assets/core-flow.png" alt="核心流程图" class="flowchart" />
<p class="image-caption">图：核心业务处理流程</p>
</div>
</div>
</section>
```
```css
.summary-text {
    font-size: clamp(0.9rem, 1.8vw, 1.2rem);
    color: var(--text-secondary);
    margin-bottom: var(--content-gap);
}
.hero-image-container {
    flex: 1;
    display: flex;
    flex-direction: column;
    align-items: center;
    justify-content: center;
    overflow: hidden;
}
```

### 策略四：卡片网格页
**适用场景：** 幻灯片包含多个并列要点或功能模块（如优势总结、功能清单）
**布局规则：**
- h2 标题：**左锚点**
- 卡片 grid 布局：`grid-template-columns: repeat(auto-fit, minmax(200px, 1fr))`
- 每行 2-4 张卡片，自适应屏幕宽度
- 卡片内文字对齐方式统一
- 卡片间距：`var(--content-gap)`
- 每个卡片最多 3 行文字
**代码示例：**
```html
<section class="slide">
<div class="slide-content">
<h2 class="reveal">方案核心优势</h2>
<div class="grid reveal">
<div class="card">
<h3>自动化程度高</h3>
<p>90% 以上的流程实现自动处理</p>
</div>
<div class="card">
<h3>处理时效快</h3>
<p>平均处理时间降低至 3 分钟</p>
</div>
<div class="card">
<h3>准确率提升</h3>
<p>自动审核准确率达到 95%</p>
</div>
</div>
</div>
</section>
```

## 图片分类呈现策略

不同的图片类型需要不同的呈现方式。根据图片内容和用途自动选择：
| 图片类型 | 识别特征 | 呈现方式 | CSS 类名 | 尺寸约束 |
|----------|----------|----------|----------|----------|
| **架构图** | 文件名含"架构/architecture"，或图片宽高比接近16:9且包含结构线条 | 全宽居中展示，带引用说明 | `.architecture` | `max-height: min(55vh, 550px)` |
| **流程图** | 文件名含"流程/flow/chart"，或图片为纵向且包含多层级节点 | 撑满剩余空间为主角 | `.flowchart` | `max-height: min(60vh, 600px)` |
| **截图/界面图** | 文件名含"截图/screenshot/界面" | 侧边展示 + 文字说明，或带边框居中展示 | `.screenshot` | `max-height: min(45vh, 400px)` |
| **数据图表** | 文件名含"数据/图表/data/chart"，或图片为图表类 | 居中展示，下方配数据来源标注 | `.chart` | `max-height: min(50vh, 450px)` |

### 图片引用/数据来源说明  使用外部引用图标或数据图表时，在图片下方用小字注明：

## 中文字体排版规范

### 字体回退顺序  所有 `font-family` 声明必须遵循：**英文字体 → 中文字体 → 通用回退**。
```css
/* 无衬线风格（现代、科技、商务） */
--font-body: "DM Sans", "Noto Sans SC", "Source Han Sans SC", "Microsoft YaHei", sans-serif;
/* 衬线风格（编辑、文学、优雅） */
--font-body: "Cormorant", "Noto Serif SC", "Source Han Serif SC", "SimSun", serif;
/* 等宽风格（代码、终端） */
--font-body: "JetBrains Mono", "Noto Sans Mono SC", "Microsoft YaHei", monospace;
```
### 行高设置
- **中文正文行高：** `1.6-1.8`（比英文的 1.4-1.5 略大，因为汉字字面更密）
- **中文标题行高：** `1.2-1.35`（标题一般较短，行高无需太大）
- 使用 CSS 变量统一管理：`--line-height-base: 1.7;`

### 首行缩进
- **禁止**使用 `text-indent` 首行缩进（汇报场景不适用，会影响阅读节奏）
- 段落之间用 `margin-bottom` 或 `gap` 分隔

### 中英文混排
- 确保字体回退链正确，英文字体在中文前
- 不需要额外的间距处理，依赖字体本身的字间距即可
- 避免在中文段落中插入不必要英文空格
